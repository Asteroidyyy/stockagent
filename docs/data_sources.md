# 数据源设计

## 目标

为第一版 A 股分析 agent 明确数据来源、用途和替换策略。

## 数据分类

### 1. 市场行情

- 个股日线 OHLCV
- 指数日线
- 板块日线

建议来源：

- MVP：AkShare
- 稳定版：Tushare Pro

### 2. 每日指标

- 换手率
- 市值
- 市盈率 / 市净率
- 涨跌幅

建议来源：

- Tushare Pro `daily_basic`

### 3. 财务数据

- 营收
- 归母净利润
- ROE
- 资产负债率

建议来源：

- Tushare Pro 财务接口

### 4. 公告与事件

- 业绩预告
- 定期报告
- 回购
- 减持
- 问询函
- 风险提示

建议来源：

- 巨潮资讯
- 上交所 / 深交所公告页面

## MVP 数据方案

第一阶段先只保留这些输入：

- 股票池日线行情
- 三大指数日线
- 行业或概念板块强弱
- 关键公告事件

这样足够支持：

- 收盘日报
- 趋势与板块判断
- 基础仓位建议

当前代码状态：

- `MockMarketDataProvider`：本地演示和联调
- `AkshareMarketDataProvider`：已接入个股日线快照、均线和趋势分
- `TushareMarketDataProvider`：已接入日线行情和核心指数快照
- `AkshareCsi500UniverseLoader`：已接入中证 500 成分股加载
- AkShare 模式下指数、板块、公告真实接入已可用
- Tushare 模式下行业板块快照暂未接入

## 采集频率

- 收盘后主任务：每日 15:30 以后
- 公告补采：每日多次
- 周末任务：重新同步财务和基准数据

## 质量问题

需要重点防止：

- 复权口径不一致
- 股票代码映射不统一
- 停牌/ST 状态遗漏
- 公告重复抓取
- 时间戳和交易日对不齐
