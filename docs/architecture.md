# 架构设计

## 1. 目标

第一版系统只做一件事：在每日收盘后，为股票池和持仓生成结构化分析与仓位建议。

## 2. 核心流程

1. 采集数据
2. 标准化和清洗
3. 生成特征
4. 应用规则与评分
5. 汇总成日报输入
6. 交给 LLM 生成可读解释
7. 保存结果并输出

## 3. 模块划分

### `data`

- 市场行情采集
- 指数与板块采集
- 公告和新闻采集
- 数据标准化

### `universe`

- 股票池定义
- 指数成分股加载
- 自选股与持仓合并
- 候选池规模控制

### `features`

- 技术指标
- 趋势强弱
- 波动与回撤
- 板块相对强度
- 事件标签

### `strategy`

- 风险过滤
- 评分引擎
- 仓位建议
- 候选池筛选

### `agent`

- 拼装上下文
- 调用 LLM
- 生成结构化日报解释

### `storage`

- 保存行情快照
- 保存公告事件
- 保存分析结果
- 保存历史日报

### `api`

- 查询最新日报
- 查询持仓建议
- 查询股票分析明细

## 4. 决策逻辑

决策采用三层结构：

1. 过滤层
   - ST、停牌、重大风险标的过滤
2. 评分层
   - 趋势分
   - 板块分
   - 基本面分
   - 事件分
   - 风险扣分
3. 解释层
   - LLM 把结构化结果写成人可读的日报

## 5. 第一版股票池

建议先从以下范围选一个：

- 自选股 + 持仓
- 沪深 300
- 中证 500

第一版不建议直接覆盖全市场。

## 6. 数据源策略

### MVP

- AkShare：行情、指数、板块
- 巨潮资讯：公告检索和补充
- OpenAI API：解释层

### 稳定版

- Tushare Pro：日线、财务、每日指标
- 交易所/巨潮：公告与风险事件

## 7. 输出结构

日报建议保留结构化字段，便于后续回测。

- `market_regime`
- `sector_summary`
- `portfolio_actions`
- `watchlist`
- `risk_alerts`
- `cash_exposure_target`

## 8. 需要准备的凭证

- `OPENAI_API_KEY`
- `TUSHARE_TOKEN`（如果启用）
- 数据库连接串
- Redis 连接串

## 9. 非目标

第一版不包含：

- 自动下单
- 高频和盘口策略
- 全市场分钟级监控
- 自训练大模型
